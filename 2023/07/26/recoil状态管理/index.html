  • 这两种方式都很难将组件树的叶子节点（使用状态的地方）与组件树的顶层（状态必须存在的地方）进行代码分拆。

recoil

核心概念

  • Atoms atom 是状态的单位，组件可以订阅、更新这些状态。当一个 atom 发生变化，每个订阅它的组件都会根据最新的状态进行重新渲染。
  • Selectors selector 是一个纯函数，用于计算基于状态的派生数据。它接收 atoms 或其他 selectors 做为入参，当上游状态发生变化时，selector 将进行重新计算。跟 atom 一样，订阅 selector 的组件也会按新状态进行重新渲染。

  1. 订阅更新状态(api)

状态创建好了，下一步就是如何在组件中订阅、更新该状态了。这里需要用到 Recoil 提供的自定义 hook，先简单介绍 3 个常用的：

  • useRecoilState 当组件同时需要读写状态时使用
  • useRecoilValue 当一个组件需要在不写入 state 的情况下读取 state 时使用
  • useSetRecoilState 当一个组件需要写入而不需要读取 state 时使用

从使用角度来看，useRecoilState 和 React 自带的 useState 用法完全一样，而useRecoilValue 和 useSetRecoilState 分别是只读和只写两种情况的对应版本。
